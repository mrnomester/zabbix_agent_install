# Zabbix Agent Installation and Host Registration Script

## Описание
Этот PowerShell скрипт автоматизирует установку Zabbix агента на Windows-хосты и их регистрацию на сервере Zabbix. Скрипт выполняет следующие действия:
1. Устанавливает Zabbix агент (если он не установлен)
2. Проверяет наличие хоста на сервере Zabbix
3. Регистрирует хост на сервере Zabbix при необходимости

## Требования
- Windows PowerShell 5.1 или новее
- Доступ к серверу Zabbix (API)
- Доступ к сетевой папке с дистрибутивом Zabbix агента
- API-токен Zabbix с необходимыми правами

## Параметры конфигурации
Перед запуском скрипта проверьте следующие параметры в начале файла:
- ZBX_SERVER = "" (IP-адрес сервера Zabbix)
- ZBX_API = "http://$ZBX_SERVER/zabbix/api_jsonrpc.php" (URL API Zabbix)
- ZBX_TOKEN = "" (API токен)
- ZBX_TEMPLATE_NAME = "Windows by Zabbix agent" (Имя шаблона, можно оставить так)
- ZBX_HOSTGRP_NAME = "Unassigned" (Имя группы хостов (не забудьте создать на сервере или указать другую группу))

$AGENT_MSI = "\\nas\Distrib\Zabbix\zabbix_agent-7.2.5.msi" # Путь к MSI-файлу агента (проверьте чтобы все целевые ПК имели доступ на выполнение)
$AGENT_PORT = "10050"  # Порт агента

## Логирование
Скрипт ведет журналы работы:
1) C:\Windows\Temp\zabbix_script.log (на каждом ПК)
2) Критические ошибки записываются в журнал событий Windows (Application).

Запуск скрипта
1) Сохраните скрипт
2) Запустите PowerShell от имени администратора
3) Выполните команду: ".\agent_install+host_create.ps1" (или укажите полный путь до скрипта)
   
## Особенности работы
1) Скрипт временно включает выполнение PowerShell скриптов только для текущего процесса, но это не факт. Лучше поставить "Set-Executionpolicy bypass"
2) Проверяется наличие уже установленного агента (по службам "Zabbix Agent" или "Zabbix Agent 2")
3) Хост регистрируется только если он еще не существует на сервере Zabbix
4) В случае ошибок скрипт завершается с кодом 1

## Безопасность
- Убедитесь, что API токен имеет минимально необходимые права
- Храните скрипт в защищенном месте
- После использования рассмотрите возможность отзыва или ограничения прав использованного API токена, регулярно меняйте токен во избежании утечки

# Автоматизация развертывания в домене
- Создаем GPO на сервере AD, назначем нужные OU и прочие права. Создаем Сценарий запука со следующими параметрами:  
  Имя сценария: %windir%\System32\WindowsPowerShell\v1.0\powershell.exe  
  Параметры сценария: -Noninteractive -ExecutionPolicy Bypass -Noprofile -file "ваш путь до скрипта"  

Версия Zabbix агента
- Скрипт использует версию агента: 7.2.5
- При необходимости обновите путь к MSI-файлу и параметры установки.
